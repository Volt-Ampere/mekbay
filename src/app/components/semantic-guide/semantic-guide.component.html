Basic Syntax

Combine text search with filters: text field=value

crab pv=20-40 → Units with "crab" in chassis or model, PV between 20-40

Operators

=Include / Equal
==Exclusive include - dropdown and semantic text fields only
!=Exclude / Not equal
>Greater than
>=Greater than or equal
<Less than
<=Less than or equal
&=AND (require all) - dropdowns and text searches only

Range Filters (Numeric)

pv=30→ Exactly 30
pv=20-40→ Between 20 and 40
pv>=20 pv<=40→ Same as above
pv!=25→ Exclude 25
pv=20-40 pv!=30→ 20-40 but not 30

Dropdown Filters

type=BM→ Meks only
type=BM,CV→ Meks OR Combat Vehicles
faction=="Draconis Combine"→ Must match Draconis Combine and no other faction
type!=BA→ Exclude Battle Armor
faction="Draconis Combine"→ Use quotes for spaces

Boolean Filters

canon:yes→ Canon units
canon:no→ Non-canon units
published:yes→ Has a published record sheet
published:no→ No published record sheet

Aliases: Use yes, true, or y for positive values; use no, false, or n for negative values.

Alpha Strike Specials

specials can match abilities or sub-abilities inside TUR(...).

specials=FLK1/1/1→ Matches FLK1/1/1, including inside TUR(...)
specials=CAR>2→ CAR value greater than 2
specials=FLK2/>2→ FLK short value 2 and medium value greater than 2
specials=FLK*/<=2→ Any FLK short value with medium value 2 or less
specials=CAR[2,4]→ CAR value 2 or 4
specials=FLK0*→ Exact FLK zero-star value (included also by >0)

Wildcards

Use * as wildcard in any dropdown filter value:

pack=*→ All units that exist in any Force Pack
faction="Free Worlds*"→ Free Worlds League and similar
faction=="Free Worlds*"→ Matches only Free Worlds-aligned factions and excludes mixed-faction entries
faction!="Free Worlds*"→ Exclude Free Worlds League and similar
era=*Republic→ Any era ending with "Republic"
equipment=AC*→ Any AC (AC2, AC5, etc.)
equipment=*Laser*→ Any equipment with "Laser"
specials=AC*/4/*→ Any AC special with a medium damage of 4
subtype=*Omni*→ All Omni subtypes

Tip: Wildcards support = (include), == (exclusive include), != (exclude), and &= (require all).

Virtual Keys

Combine related filters with slash notation:

dmg=3/2/1→ dmgS=3, dmgM=2, dmgL=1
dmg=*/2/*→ dmgM=2 (any short/long)
dmg=*/*/*/3→ dmgE=3 (any short/medium/long)

Linked Filters (Alpha Strike)

The motive and mv filters work together. When both are active, movement value filtering only applies to the selected motive types:

motive=j→ Units with Jump movement (or use motive=jump)
mv=8-12→ Units with any movement value between 8-12
motive=j mv>=6→ Units with Jump movement of 6+
motive=w,t mv=4-8→ Units with Wheeled or Tracked movement between 4-8

Tip: Motive accepts codes (j, w, t, h, v, g) or full names (Jump, Wheeled, Tracked, Hover, VTOL, WiGE).

Note: Without a motive filter, mv checks any movement mode. With a motive filter, only the selected modes are checked.

Boolean Logic & Grouping

Use OR and AND keywords with parentheses for complex queries:

type=BM pv>50→ Meks AND pv>50 (implicit AND)
crab or ares→ all units with "chassis" or "model" containing Crab OR Ares
(type=BM pv>50) OR (type=CV pv<10)→ Meks with pv > 50 OR Vehicles with pv < 10

Note: Filters at the same level are AND'd by default. Use OR to combine groups.

Escape Sequences

Use backslash \ to include special characters literally in text search:

Puma \(Adder\)→ Search for "Puma (Adder)" with literal parentheses

Escapable characters: ( ) = > < ! & " ' \

Text Search Fields (Semantic Only)

These fields are only available via text input (no UI panel). They use exact match by default, unlike the fuzzy text search. Use * wildcards for partial matching:

@for (f of semanticFilters(); track f.key) {
{{ f.key }} {{ f.label }}
}
chassis=Crab→ Exact match: only "Crab" chassis
chassis=*Crab→ Ends with "Crab" (Crab, Hermit Crab, King Crab)
chassis=*Crab*→ Contains "Crab" anywhere
chassis&=*Crab*→ Must contain "Crab" (AND logic for multiple)
chassis!=*Crab*→ Excludes anything containing "Crab"
chassis=*Crab chassis!=Hermit*→ Contains "Crab" but excludes "Hermit"
mul=5,10,11→ Units with MUL ID 5, 10, or 11
model=*Prime→ Prime variants only
model!=*Prime→ Exclude Prime variants

Note: chassis and model match exact field values, while plain text search (e.g., crab) does fuzzy matching across both fields.
